# PRD: Prompt Library, Report Generation, White-Label Branding & Onboarding

**Project:** OMG Bridge
**Date:** 2026-05-10
**Owner:** WODO
**Status:** Ready for implementation

---

## 1. Overview

This PRD adds four interconnected features to OMG Bridge:

1. **Prompt Library** — A curated + user-customizable library of prompt templates in the dashboard, with semantic tagging, search/filter, and copy-to-clipboard
2. **Report Generation System** — A base HTML report template with data cards, tabs, and commentary sections, using CSS variables for branding, with a print-to-PDF button
3. **White-Label Branding** — User uploads their logo, dominant colors are extracted and shown for approval, stored as a brand profile, and injected into report generation prompts
4. **Onboarding Wizard** — A full-screen step-by-step flow for new users (property selection → branding setup → MCP URL) with plan-aware upgrade prompts

Additionally:
- Two new MCP tools (`list_report_templates` and `get_report_template`) for in-chat prompt discovery
- Admin dashboard prompt management (add/edit/delete system-wide prompts)
- Upgrade flow for free users (banners at 50 and 180 of 200 tool calls)

## 3. Prompt Library

### 3.1 Prompt Template Structure

Each prompt template (system or user) has this structure:

```typescript
interface PromptTemplate {
  id: string;
  title: string;           // "Monthly SEO Performance Report"
  description: string;     // "Comprehensive monthly report covering organic traffic, keyword rankings, and page performance"
  body: string;            // The full prompt text (see 3.2)
  category: string;        // "seo-report" | "traffic-analysis" | "aeo" | "competitor" | "technical-seo" | "custom"
  requiredConnections: string[]; // ["gsc", "ga4"] or ["gsc", "ga4", "gbp"]
  questions: string[];     // Clarifying questions baked into the prompt
  semanticTags: string[];  // Auto-generated: ["organic traffic", "keyword ranking", "CTR", "monthly"]
}
```

### 3.2 Prompt Body Format

Each prompt body must follow this structure so the AI knows to ask clarifying questions before executing:

```
## {{title}}

### Before generating this report, ask the user the following questions:
{{#each questions}}
- {{this}}
{{/each}}

### Once all questions are answered, generate the report using these instructions:

{{main prompt instructions}}

### Report Format:
- Generate as a self-contained HTML file
- Use the brand profile provided below for styling (colors, logo, font)
- Include: header with logo, executive summary, data cards, tabbed sections, commentary
- Include a "Download as PDF" button that triggers window.print()

### Brand Profile:
- Company: {{brand.companyName || "OMG Bridge"}}
- Logo URL: {{brand.logoUrl || "/OMG Rectangle LOGO Dark BG.svg"}}
- Primary Color: {{brand.primaryColor || "#00B3B3"}}
- Secondary Color: {{brand.secondaryColor || "#0E1420"}}
- Font: {{brand.fontFamily || "Inter"}}
```

The `{{placeholders}}` are NOT processed server-side. They are part of the prompt text that the AI reads and interprets. The brand profile section gets populated when the prompt is copied — if the user has a brand profile, their values are injected; otherwise OMG Bridge defaults are used.

### 3.4 Semantic Auto-Tagging

When a prompt is created or updated (system or user), call the Anthropic API (Claude Haiku) server-side to generate semantic tags:

```typescript
// src/lib/prompt-tagger.ts
async function generateSemanticTags(title: string, description: string, body: string): Promise<string[]> {
  const response = await anthropic.messages.create({
    model: "claude-haiku-4-5-20251001",
    max_tokens: 200,
    system: "Extract 5-8 semantic keywords/phrases from this prompt template. Return as a JSON array of strings. Focus on the topics, metrics, and use cases the prompt covers.",
    messages: [{
      role: "user",
      content: `Title: ${title}\nDescription: ${description}\nBody: ${body.substring(0, 500)}`
    }]
  });
  return JSON.parse(response.content[0].text);
}
```

Add `ANTHROPIC_API_KEY` to `.env` and Railway environment variables. This is used server-side only for tagging.

### 3.5 Search & Filter

Search should query across: `title`, `description`, `category`, and `semanticTags`.

Use PostgreSQL full-text search:

```sql
-- Add a generated tsvector column or use a runtime query:
SELECT * FROM prompt_templates
WHERE is_active = true
AND (
  to_tsvector('english', title || ' ' || description || ' ' || semantic_tags::text)
  @@ plainto_tsquery('english', $1)
)
ORDER BY ts_rank(...) DESC;
```

For Prisma, this can be done with a raw query or by filtering in application code for v1 (the dataset is small enough).

Filter options in the UI:
- **Category pills**: All, SEO Reports, Traffic Analysis, AEO, Technical SEO, GBP, Custom
- **Connection filter**: GSC, GA4, GBP (show only prompts matching connected services)
- **Search bar**: Free-text search across title + description + tags

### 7.3 How Reports Get Generated

The flow is:
1. User copies prompt from library (or selects via MCP tool)
2. Prompt instructs the AI to ask clarifying questions
3. AI asks questions, user answers
4. AI calls OMG Bridge MCP tools (get_search_analytics, ga_run_report, etc.) to fetch real data
5. AI generates a self-contained HTML file using the template structure above
6. AI presents the HTML file to the user (in Claude, this renders as an artifact; in Claude Code/Cursor, it saves as a .html file)

The key insight: the report template is a **reference/instruction** embedded in the prompt, not a server-side template engine. The AI reads the CSS structure and generates HTML that follows it, with real data and commentary.

### 8.4 Color Extraction from Logo

Client-side using Canvas API. Create a utility component:

```typescript
// src/lib/color-extractor.ts
// 1. Load the uploaded image into an off-screen canvas
// 2. Get pixel data via getImageData()
// 3. Run a simplified k-means clustering (k=3) on the pixel colors
//    - Ignore very light pixels (near white, background)
//    - Ignore very dark pixels (near black, outlines)
// 4. Return top 3 dominant colors as hex strings
// 5. Sort by vibrancy/saturation — most vibrant = primary

export function extractColors(imageFile: File): Promise<{
  primary: string;
  secondary: string;
  accent: string;
}>;
```

The extracted colors are shown as swatches. The user can:
- Accept them as-is
- Click a swatch to open a color picker and override
- Re-upload a different logo to re-extract

### 8.6 Brand Profile in Prompts

When a prompt is copied or retrieved via MCP tool, the brand section is populated:

```typescript
function injectBrandProfile(promptBody: string, brand: BrandProfile | null): string {
  const defaults = {
    companyName: "OMG Bridge",
    logoUrl: "https://bridge.theomg.ai/OMG Rectangle LOGO Light BG.svg",
    primaryColor: "#00B3B3",
    secondaryColor: "#0E1420",
    fontFamily: "Inter",
  };

  const values = brand?.isApproved ? {
    companyName: brand.companyName || defaults.companyName,
    logoUrl: brand.logoUrl || defaults.logoUrl,
    primaryColor: brand.primaryColor || defaults.primaryColor,
    secondaryColor: brand.secondaryColor || defaults.secondaryColor,
    fontFamily: brand.fontFamily || defaults.fontFamily,
  } : defaults;

  return promptBody
    .replace("{{brand.companyName}}", values.companyName)
    .replace("{{brand.logoUrl}}", values.logoUrl)
    .replace("{{brand.primaryColor}}", values.primaryColor)
    .replace("{{brand.secondaryColor}}", values.secondaryColor)
    .replace("{{brand.fontFamily}}", values.fontFamily);
}
```

```typescript
// In layout.tsx
const subscription = await db.userSubscription.findUnique({
```
